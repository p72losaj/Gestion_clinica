PRACTICA 3

 	

	DIAGRAMAS DE SECUENCIA

    El archivo markdown de diagramas de secuencia no es lo que se pedía. Se pedía un markdown que contuviera las imágenes de todos los diagramas.

    Mensaje de error no debería ser un elemento del diagrama de secuencia. Es simplemente un mensaje.

    En general se podrían haber simplificado más los diagramas de secuencia. No se pedía detallarlos tanto. Bastaría con poner Actor, Sistema y BBDD puesto que el sistema a desarrollar es bastante sencillo.

    Los bloques alt se deben crear utilizando el bloque existente en visual paradigm para ello (ej. diagrama de modificar tratamiento).

    Todos los bloques alt, opt y loop deben tener su condición.


	DIAGRAMA DE CLASES

    Los nombres de los atributos deberían ser más descriptivos.

    Las relaciones no se indican así. Mirad las diapositivas de la práctica.

    El historial se ha definido como un atributo de tipo texto del paciente. Esto se puede hacer así pero tendréis que tener cuidado cuando hagáis la implementación porque si se permite editar como texto, es probable que no se cumplan algunos de los requisitos que se indicaron al inicio. Por ejemplo, el requisito que indicaba que no se pueden borrar entradas del historial.
